Importar Notas Fiscais

Importe dados de notas fiscais através de arquivo CSV

Selecionar Arquivo
{% csrf_token %}
Formatos aceitos: CSV (.csv) com delimitador ponto e vírgula (;)
Quando marcado, apenas registros que ainda não existem na tabela serão importados. Registros duplicados serão ignorados.
Quando marcado, registros existentes serão atualizados com os dados do arquivo. Nota: Se "Apenas adicionar novos registros" estiver marcado, esta opção será ignorada.
Instruções
Formato do Arquivo:

O arquivo CSV deve usar delimitador ponto e vírgula (;) e conter as seguintes colunas (primeira linha como cabeçalho):

  • Emitente - CNPJ do emitente (obrigatório)
  • Nome fantasia emitente - Nome fantasia do emitente
  • Nota - Número da nota fiscal (obrigatório)
  • Série - Série da nota fiscal
  • Modelo - Modelo da nota fiscal
  • Total Nota - Valor total da nota (formato brasileiro: 2.982,99)
  • Data emissão - Data de emissão da nota
  • Data vencimento - Data de vencimento
  • Ctrle - Controle
  • Unidade - Código da unidade
  • Nome unidade - Nome da unidade
  • Unidade autorização - Unidade de autorização
  • Nome unidade autorização - Nome da unidade de autorização
  • Centro atividade - Código do centro de atividade
  • Nome centro atividade - Nome do centro de atividade
  • Situação - Situação da nota (PENDENTE, LANÇADA, AUTORIZADA, AGUARDANDO AUTORIZAÇÃO)
  • Situação - Situação detalhada (segunda coluna)
  • Nome usuário - Nome do usuário
  • Data inclusão - Data de inclusão
  • Observações - Observações gerais
  • Observações CSC - Observações CSC
  • Data autorização - Data de autorização
  • Observações autorização - Observações da autorização
  • Data ult sit fechada - Data da última situação fechada
  • Autorizador - Nome do autorizador
  • LANCAMENTO TESF0028 - Lançamento TESF0028
Observações:
  • A primeira linha do arquivo deve conter os cabeçalhos das colunas
  • Os campos Emitente e Nota são obrigatórios
  • O sistema identifica registros duplicados pela combinação de Emitente + Nota + Série + Modelo
  • Linhas de resumo (que começam com "Unidade") são automaticamente ignoradas
  • O sistema aceita valores monetários no formato brasileiro (ex: 2.982,99)

Nota: O sistema suporta arquivos CSV com diferentes encodings (latin-1, iso-8859-1, utf-8, cp1252) e tenta detectar automaticamente o encoding correto.
